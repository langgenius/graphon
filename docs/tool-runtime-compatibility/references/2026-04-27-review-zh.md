# 代码审查：8428e5c

Commit: `8428e5c0ac9559ffbf40076058de9e46a947582b`

标题：`Pass tool node execution id to runtime`

## 目标推断

这个 commit 的目标是在 `ToolNode.run()` 获取 tool runtime 之前，先恢复或创建当前
node execution id，然后把这个 execution id 传给 workflow 层的 tool runtime
adapter。

同时，这个 commit 也试图兼容尚未接收 `node_execution_id` 参数的旧 runtime
adapter。

## 审查范围

- 是否符合 SOLID 原则，以及职责划分是否清晰。
- 在适用场景下是否使用强类型和 schema，而不是无结构的 map。
- 是否保持向后兼容，特别是对现有 workflow runtime adapter 和历史 tool node 数据的兼容。

## 问题列表

### P1：保留 legacy `variable_pool` 行为

文件：`src/graphon/nodes/tool/tool_node.py:115`

这个 commit 的目标是把 node execution id 传给 runtime，但它同时删除了一段历史兼容
逻辑：当 legacy tool node 满足 `version == "1"` 且 `tool_node_version is None`
时，旧代码会传入 `variable_pool=None`。

被删除的注释明确说明，这个分支是为了向后兼容而保留的。现在总是传入
`graph_runtime_state.variable_pool`，可能会破坏那些仍然依赖 `None` 来触发 legacy
参数处理逻辑的 adapter。

建议修复：保留旧的 `variable_pool` 选择逻辑，并独立传递 `node_execution_id`。

### P2：避免掩盖 runtime 内部的 `TypeError`

文件：`src/graphon/nodes/tool/tool_node.py:259`

如果签名检查已经确认 runtime 接收 `node_execution_id`，当前代码仍然会捕获
`TypeError`，并在错误信息包含 `node_execution_id` 和 `unexpected keyword` 时去掉
该参数重试。

这可能会掩盖 runtime 内部真实的 bug。例如，某个 runtime 明确支持该参数，但在内部转发
keyword arguments 时写错了；当前 fallback 会吞掉这个错误，还可能导致 runtime 状态被初始化两次。

建议修复：把兼容分支拆开。只有当签名声明支持 `node_execution_id` 时才带上该参数调用；
当签名不支持时才不带该参数调用。对于明确兼容新参数的 runtime，不应在进入 runtime 后再 fallback。

### P3：移除未使用的 import

文件：`src/graphon/nodes/tool/tool_node.py:4`

`ruff` 报告 `F401`，因为 `TYPE_CHECKING` 被导入但没有使用。

建议修复：合并前移除这个未使用的 import。

### P3：恢复顶层函数间距

文件：`tests/nodes/tool/test_tool_node.py:111`

`ruff` 报告 `E302`，因为顶层函数 `_build_tool_node` 前只有一个空行。

建议修复：补上缺少的空行。

## 强类型说明

这个 commit 没有明显新增需要 Pydantic schema 的业务数据 map。现有的
`build_file_reference(mapping=...)` 协议仍然使用 `Mapping[str, Any]`，但这是既有行为，
不是这个 commit 引入的主要风险。

相比继续增加无结构的 runtime inspection，runtime 兼容逻辑更适合通过明确的 protocol
capability 或更窄的 adapter interface 来表达。

## 结论

是否可以合并：不建议。

传递 execution id 这个目标本身是合理的，测试也覆盖了 happy path 和一种 legacy adapter
形态。但是当前 commit 同时引入了一个与目标不直接相关的 `variable_pool` 兼容性回归，
并且 fallback 路径可能掩盖 runtime adapter 内部 bug。建议先修复 P1 和 P2，再清理两个
lint 问题。
